# Simplify where possible
Replaced my portfolio website from dynamic wordpress using hostinger hosting to a static one hosted for free on github. 
> The major portion of my portfolio was blogging, reset everything was almost static. With the limitation of wordpress, the frequent backups and
hosting price, I shifted to check if static website using github pages was useful. Created a template page for my blogs with standard components. 
Most of the website is static. For adding blog, I reuse the templates page to create a new blog. Place it in a folder and creat and entry in blogs
page which has links to all my blogs. I now pay only for my domain, host everything on github pages, take easy zip backups, easy blogs creation, no
database and forever platform independence.

# Code Reading will be valuable skill
While AI has disrupted and made coders restless, the probability layer that has even as minute as 0.00001% wrong result possibility, engineers would 
still be needed to cater to reading code and validating logically.
> Before this development was working on logical layer, the compilers work on logic, the frameworks work on logic. But now the AI layer is the first layer to work on probability at its core (neural network). So it cant be used directly
in systems where business needs acccuracy and rely on absolute logic. Coder readers with understanding of logic, design principles, arhitecture, business,
domain and a backing of experience will always be a valueable asset. Imagine you were owner of a payment application, would you trust AI for accuracy, 
even if it has reached 99.9999999% accuracy. For some their application is their life and putting AI to work on it without human assistance is life threatning.
